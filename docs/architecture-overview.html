Layer by Layer

1. Data Ingestion (where it all starts)

Exercise 09

FRED API → Raw Data

Fetches daily Treasury par yields from the Federal Reserve. This is the raw material everything else is built from.

Exercise 10

S3 Storage

Stores the raw JSON files. Versioned, durable, cheap. The "lake" where raw data lands before processing.

Exercise 11

RDS PostgreSQL

Structured storage with tables, columns, and relationships. Where the data goes after being cleaned and organized.

2. Curve Building (turning raw data into useful curves)

Exercise 12

Spot Curve Bootstrapping

Takes par yields and extracts the "pure" zero-coupon rates at each maturity. The foundation for everything downstream.

Exercise 13

Forward Rate Curve

Derives implied future rates from the spot curve. "What does the market expect 1-year rates to be in 5 years?"

Exercise 14

Nelson-Siegel / Svensson Fitting

Fits a smooth parametric curve through observed yields. 4-6 numbers that describe the entire term structure.

3. Credit Risk (measuring and pricing default risk)

Exercise 15

Credit Spreads + CDS

Corporate yield minus Treasury yield = credit spread. CDS spreads → hazard rates → default probabilities.

Exercise 18

Default Probability Term Structure

Piecewise hazard rates: different default intensity at different horizons. Survival curves and expected loss.

Exercise 19

OAS / Z-Spread (Binomial Tree)

Pricing callable bonds with a rate tree. OAS strips out the call option to isolate pure credit spread.

Exercise 20

Portfolio Credit VaR

Monte Carlo simulation with correlated spread shocks. "How bad could a portfolio loss get in the worst 5% of scenarios?"

4. Messaging & Real-Time (making it event-driven)

Exercise 15

SQS Queues + SNS Alerts

SQS: async job queue ("recalculate BBB spreads"). SNS: fan-out alerts ("BBB spreads just spiked, email the team").

Exercise 16

WebSocket Streaming

Push live spread updates to connected dashboards. No polling — server sends data the instant it changes.

5. Performance & Caching

Exercise 17

Redis / ElastiCache

Cache expensive calculations (curve fitting, pricing). First request computes; subsequent requests get instant answers. Falls back gracefully if Redis is down.

6. Infrastructure & Operations

Exercise 13

GitHub Actions CI/CD

Push code → tests run automatically → deploys to AWS if they pass. No manual deployment steps.

Exercise 14, 18

Terraform

Infrastructure as code. Describe what you want in a file, Terraform builds it. Reproducible, versionable, destroyable.

Exercise 19

CloudWatch Monitoring

Metrics, dashboards, alarms. Knows when things break before users notice. "API error rate > 5% → page someone."

The key idea: Each exercise builds one piece. The capstones snap them together like LEGO bricks. Capstone A is the "rates" engine (Treasury curves, fitting, pricing). Capstone B is the "credit" engine (spreads, defaults, risk) that uses A's curves as inputs.
